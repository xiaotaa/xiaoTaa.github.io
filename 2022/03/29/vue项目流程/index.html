创建vue项目

vue create 项目名

补充

  1. node_modules文件夹: 项目依赖文件夹

  2. public文件夹：一般放置静态资源(图片),需要注意,webpack打包时这里面的静态资源会原封不动的打包到dist文件夹中

  3. src文件夹(我们写的项目文件会放在这里面)：

    3.1 assets文件夹：一般也放静态资源(一般放置多个组件公用的静态资源),需要注意,webpack打包时会把这里的静态资源当成一个模块，打包到js文件里面

    3.2 components文件夹: 一般放置的是非路由组件(全局组件)

    3.3 app.vue: 唯一的根组件

    3.4 main.js: 程序入口文件,也是整个项目当中最先执行的文件

  4. babel.config.js：配置文件(Babel相关)

  5. package-lock.json: 缓存性文件

项目的其他配置

运行项目网页可自动打开

—package.json-> “scripts”中修改

“serve”: “vue-cli-service serve –open”

eslint校验功能关闭

— 在根目录创建”vue.config.js”文件

module.exports = {
   // 关闭eslint
   lintOnSave: false
 }

配置src别名

— 根目录”jsconfig.json”文件中

{
  // 配置别名，在node_modules dist中不适用
  "compilerOptions": {
    "baseUrl": "./",
    "paths": { 
      "@/*": ["src/*"]
    }
  },
  "exclude":["node_modules","dist"]
}

less样式

  1. 需要注意浏览器不识别less样式，需要通过less,less-loader进行处理,把less样式变为css样式

    npm install –save less less-loader@5 //安装less、less-loader(版本5)

  2. 如果想让组件识别less样式，需要在style标签加上lang=less

路由组件的搭建

- components文件夹: 非路由组件(共用的全局组件)
- pages|views文件夹: 放置路由组件(自己写的页面)

路由的搭建

  1. 安装路由

    npm install vue-router

  2. — router->index.js文件
    ```
    // 引入vue和vue-router
    import Vue from “vue”
    import VueRouter from “vue-router”
    // 使用插件
    Vue.use(VueRouter)

// 引入路由组件
import Home from “@/views/Home/index.vue”
// 暴露一个VueRouter实例
export default new VueRouter({
// 配置路由
routes:[
{
path:”/home”,
component: Home
},
// 重定向
{
path: “*”,
redirect:”/home”
}
]
})

3. --- main.js

//引入路由 +++
import router from “@/router”
new Vue({
render: h => h(App),
// 注册路由 +++
router
}).$mount(‘#app’)

如果这步出错，要检查下是否是路由版本的问题,vue2的话路由版本要用3的版本

### 路由组件跟非路由组件的区别
1. 路由组件通常在pages|views文件夹中,非路由组件通常在components中
2. 路由组件一般要在router文件中注册,使用的时候是用组件的名字;非路由组件使用的时候是用标签
3. 注册完路由,不管是路由组件还是非路由组件都有$route、$router属性

$route: 一般获取路由信息[路径,query,params]

$router: 一般进行编程式导航进行路由跳转[push,replace]

push跟replace的区别就是有没有路由跳转的历史记录

## 路由传参
### 1. 路由跳转有几种方式？
声明式导航: router-link (务必要有to属性)
编程式导航: 利用的是组件实例的$router.push|replace方法

### 2. 路由传参,参数有几种写法
params参数: 需要注意，在配置路由的时候，需要占位

query参数: 类似于Ajax中的queryString  /home?k=v&k=v,不需要占位

注意: params参数要配合路由的name属性使用,不能使用path属性

注意：
配置路由的写法
path:”/search/:keyword”, //使用了占位符,使用params传递参数时要用到
name:”search”,
component: Search,
路由传递参数的写法：
1.字符串形式
this.$router.push(“/search/“ + this.keyword+ “?v=” + this.keyword.toUpperCase());
2.模板字符串形式
this.$router.push(/search/${this.keyword}?v=${this.keyword.toUpperCase()})
3.对象
this.$router.push({name:”search”,params:{keyword:this.keyword},query:{k:this.keyword.toUpperCase()}})

getters: 相当于计算属性

// vuex需要使用一次

// state: 存储数据的仓库

三级联动节流

import _ from "lodash"; //全部引用
import throttle from "lodash/throttle.js";  //推荐(按需引用)

methods:{
    //注意：throttle回调函数别用箭头函数，可能会出现上下文this问题
    changBack: throttle(function(index){
         // index:鼠标移上某一个一级分类的元素的索引值
        // 正常情况(用户慢慢的操作)：鼠标进入，每一个一级分类h3，都会触发鼠标进入事件
        // 非正常情况(用户操作很快)：本身全部的一级分类都应该触发鼠标进入事件，但经过测试，只有部分h3触发了
        // 就是由于用户行为过快，导致浏览器反应不过来。如果当前回调函数中有一些大量业务，有可能出现卡顿现象
        // 解决办法：防抖和节流
        // 节流：在规定的间隔事件范围内不会重复触发回调，只有大于这个时间间隔才会触发回调，把频繁触发变为少量触发
        // 防抖：前面所有的触发都被取消，最后一次执行在规定的时间之后才会触发，也就是说如果连续快速的触发，只会执行一次
        // 可用延时器，或者lodash插件的debounce函数(防抖)throttle函数(节流)
          this.curIndex = index;
      },50),
}

三级联动路由跳转跟传递参数

跳转方式：
声明式：router-link
编程式：push | replace (this.$router.push)

如果使用声明式跳转，三级联动会出现卡顿现象,因为router-link是组件

最好的解决办法是： 编程式 + 事件委托

利用事件委托会出现问题：1. 点击是a标签 2. 获取参数 （1、2、3级的参数）

解决：通过自定义属性定位到a标签，event.target.dataset 获取自定义属性和属性值

– TypeNav

//在最近的父元素上绑定单击事件，在a标签上添加自定义属性
<a
  :data-categoryName="c1.categoryName"
  :data-category1Id="c1.categoryId"
 >
    {{ c1.categoryName}}
</a>

goSearch(event) {
      console.log(event.target);
      // 可通过event.target.dataset获取自定义属性跟属性值
      // this.$router.push(name:{跳转的路径},query:{参数})
      let {
        categoryname,
        category1id,
        category2id,
        category3id
      } = event.target.dataset;
      if (categoryname) {
        let location = { name: "serach" };
        let query = { categoryName: categoryname };
        if (category1id) {
          query.category1Id = category1id;
        } else if (category2id) {
          query.category2Id = category2id;
        } else {
          query.category3Id = category3id;
        }
        location.query = query;
        this.$router.push(location)
      }
    }

三级联动请求优化

原请求接口在TypeNav组件中，切换页面时会再次触发接口请求

优化：跟组件app.vue只会触发一次，在使用三级联动之前就已经获取了数据，main.js也只触发一次，但main.js不是一个组件，所以不能写在main.js中（this为undefined）
