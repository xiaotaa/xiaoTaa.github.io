# 创建 vue 项目

vue create 项目名

# 补充

  1. node_modules 文件夹：项目依赖文件夹

  2. public 文件夹：一般放置静态资源 (图片), 需要注意，webpack 打包时这里面的静态资源会原封不动的打包到 dist 文件夹中

  3. src 文件夹 (我们写的项目文件会放在这里面)：

    3.1 assets 文件夹：一般也放静态资源 (一般放置多个组件公用的静态资源), 需要注意，webpack 打包时会把这里的静态资源当成一个模块，打包到 js 文件里面

    3.2 components 文件夹：一般放置的是非路由组件 (全局组件)

    3.3 app.vue: 唯一的根组件

    3.4 main.js: 程序入口文件，也是整个项目当中最先执行的文件

  4. babel.config.js：配置文件 (Babel 相关)

  5. package-lock.json: 缓存性文件

# 项目的其他配置

# 运行项目网页可自动打开

---package.json-> "scripts" 中修改

"serve": "vue-cli-service serve --open"

# eslint 校验功能关闭

--- 在根目录创建 "vue.config.js" 文件

module.exports = {
   // 关闭eslint
   lintOnSave: false
 }

# 配置 src 别名

--- 根目录 "jsconfig.json" 文件中

{
  // 配置别名，在node_modules dist中不适用
  "compilerOptions": {
    "baseUrl": "./",
    "paths": { 
      "@/*": ["src/*"]
    }
  },
  "exclude":["node_modules","dist"]
}

# less 样式

  1. 需要注意浏览器不识别 less 样式，需要通过 less,less-loader 进行处理，把 less 样式变为 css 样式

npm install --save less less-loader@5 // 安装 less、less-loader (版本 5)

  1. 如果想让组件识别 less 样式，需要在 style 标签加上 lang=less

# 路由组件的搭建

- components文件夹: 非路由组件(共用的全局组件)
- pages|views文件夹: 放置路由组件(自己写的页面)

# 路由的搭建

  1. 安装路由

    npm install vue-router

  2. --- router->index.js 文件

// 引入vue和vue-router
import Vue from "vue"
import VueRouter from "vue-router"
// 使用插件
Vue.use(VueRouter)

// 引入路由组件
import Home from "@/views/Home/index.vue"
// 暴露一个VueRouter实例
export default new VueRouter({
  // 配置路由
  routes:[
    {
      path:"/home",
      component: Home
    },
    // 重定向
    {
      path: "*",
      redirect:"/home"
    }
  ]
})
  1. --- main.js
//引入路由 +++
import router from "@/router"
new Vue({
  render: h => h(App),
  // 注册路由 +++
  router
}).$mount('#app')

如果这步出错，要检查下是否是路由版本的问题，vue2 的话路由版本要用 3 的版本

# 路由组件跟非路由组件的区别

  1. 路由组件通常在 pages|views 文件夹中，非路由组件通常在 components 中
  2. 路由组件一般要在 router 文件中注册，使用的时候是用组件的名字；非路由组件使用的时候是用标签
  3. 注册完路由，不管是路由组件还是非路由组件都有route、route、route、router 属性

$route: 一般获取路由信息 [路径，query,params]

$router: 一般进行编程式导航进行路由跳转 [push,replace]

push 跟 replace 的区别就是有没有路由跳转的历史记录

# 组件的显示与隐藏

v-if || v-show

通过路由元信息也可以做到组件的显示与隐藏

 routes:[
    {
      path:"/home",
      component: Home,
      // 路由元信息，meta字段
      meta:{
        footerShow: true
      }
    },
    {
      path:"/login",
      component: Login,
      // 路由元信息，meta字段
      meta:{
        footerShow: false
      }
    },
 ]

//显示与隐藏的组件
<Footer v-show="$route.meta.footerShow"></Footer>

# 路由传参

# 1. 路由跳转有几种方式？

声明式导航: router-link (务必要有 to 属性)
编程式导航：利用的是组件实例的 $router.push|replace 方法

# 2. 路由传参，参数有几种写法

params 参数：需要注意，在配置路由的时候，需要占位

query 参数：类似于 Ajax 中的 queryString /home?k=v&k=v, 不需要占位

注意: params 参数要配合路由的 name 属性使用，不能使用 path 属性

 注意：
    配置路由的写法
             path:"/search/:keyword",       //使用了占位符,使用params传递参数时要用到
             name:"search",
             component: Search,
 路由传递参数的写法：
      1.字符串形式
      this.$router.push("/search/" + this.keyword+ "?v=" +  this.keyword.toUpperCase());
      2.模板字符串形式
      this.$router.push(`/search/${this.keyword}?v=${this.keyword.toUpperCase()}`)
      3.对象
      this.$router.push({name:"search",params:{keyword:this.keyword},query:{k:this.keyword.toUpperCase()}})
