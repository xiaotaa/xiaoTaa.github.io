搭建vue项目时遇到的问题

发表于 2021-06-11
  • xiao Taa
首页

# 路由问题


原因：没有在 main 文件里加载 router 文件

解决方法:

import router from './router' //引入router/index.js文件

new Vue({
  router,   //
  render: h => h(App),
}).$mount('#app')

# vue 项目打包到服务器报错 net::ERR_ABORTED 404 (Not Found)


解决方法:
新建一个 vue.config.js 文件，复制下面的代码

module.exports = {
  publicPath: "./"
}

# 登陆时没带上 cookie，导致登录失败

原文章：https://blog.csdn.net/lovelyelfpop/article/details/108501677

原因：谷歌浏览器 Chrome80+ 开启了 SameSite="Lax"（限制跨站访问 Cookie）

解决方法：

  1. (现在这个项目中用的这个)
  • 在本地 hosts 文件（C:\WINDOWS\system32\drivers\etc）
    添加 127.0.0.1 c.zhaohu.co
  • 在项目 vue.config.js 文件中找到 host
    修改为 host: "c.zhaohu.co"

  1. 开发环境
    可以从浏览器进入 chrome://flags/#same-site-by-default-cookies ，将 SameSite by default cookies 设为 disabled

  2. 生产环境
    使用反向代理，将不同站点代理成同一域名。
    不存在跨站，就没有这个问题了

# 重复的 Tag


原因： 菜单栏加载了两遍
解决方案： 每次加载前，都将菜单栏清空
