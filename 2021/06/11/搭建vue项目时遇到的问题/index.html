# 路由问题


原因：没有在 main 文件里加载 router 文件

解决方法:

import router from './router' //引入router/index.js文件

new Vue({
  router,   //
  render: h => h(App),
}).$mount('#app')

# vue 项目打包到服务器报错 net::ERR_ABORTED 404 (Not Found)


解决方法:
新建一个 vue.config.js 文件，复制下面的代码

module.exports = {
  publicPath: "./"
}

# 登陆时没带上 cookie，导致登录失败

原文章：https://blog.csdn.net/lovelyelfpop/article/details/108501677

原因：谷歌浏览器 Chrome80+ 开启了 SameSite="Lax"（限制跨站访问 Cookie）

解决方法：

  1. (现在这个项目中用的这个)
  • 在本地 hosts 文件（C:\WINDOWS\system32\drivers\etc）
    添加 127.0.0.1 c.zhaohu.co
  • 在项目 vue.config.js 文件中找到 host
    修改为 host: "c.zhaohu.co"

  1. 开发环境
    可以从浏览器进入 chrome://flags/#same-site-by-default-cookies ，将 SameSite by default cookies 设为 disabled

  2. 生产环境
    使用反向代理，将不同站点代理成同一域名。
    不存在跨站，就没有这个问题了

# 编程时路由跳转至当前页面 (参数不变)，多次执行会报 NavigationDuplicated 错

-- 路由跳转有两种形式：声明式导航、编程式导航
-- 声明式导航没有这类问题，是因为 vue-router 底层已经处理好了

解决：

  1. 通过给 push 方法传递成功和失败的回调函数 (治标不治本)

eg:this.$router.push({name:"",params:{k:v},query:{k:v}},()=>{},()=>{})

  1. 重写方法
    this: 当前组件实例
    this.router属性:这个属性是当前属性值VueRouter类的一个实例,当在入口文件注册路由时,给组件实例添加router属性: 这个属性是当前属性值VueRouter类的一个实例, 当在入口文件注册路由时,给组件实例添加router属性:这个属性是当前属性值VueRouter类的一个实例,当在入口文件注册路由时,给组件实例添加 route|$router 属性
    push: 是 VueRouter 类的一个原型方法

类的实例可以直接调用类的原型方法

   // 重写push方法就是在重写之前保存一下原来的push方法，在重写的方法中调用保存的push方法，但是因为我们保存的push是挂载在window上的方法，所以要通过call修改this的指向

   // 先把VueRouter原型的push方法保存一下
      let originPush = VueRouter.prototype.push;
   // 重写push || replace方法
   // call跟apply的区别：
   // 相同：都可以调用函数一次，更改篡改函数的上下文一次
   // 不同：call的参数用逗号隔开，apply传递的参数数组，传递多个参数的时候用apply
       VueRouter.prototype.push = function (loction, resolve, reject) {
       if (resolve && reject) {
           originPush.call(this, loction, resolve, reject)
       } else {
           originPush.call(this, loction, () => { }, () => { })
       }
       }
